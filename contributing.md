# Contributing — Adding New Examples

Thanks for contributing to requiems-api-tutorials. This guide covers everything you need to add a new example project.

---

## What Makes a Good Example

- **Focused** — demonstrates one API feature, not many
- **Runnable in under 5 minutes** from a fresh clone
- **Minimal boilerplate** — only what the framework requires to function
- **Real output** — shows actual API responses, not mocked data
- **Clear README** — someone unfamiliar with the API can follow it without help

---

## Directory Naming

```
{framework}-{api-feature}
```

Lowercase, hyphenated. Examples:

```
nextjs-email-validation
fastapi-qr-code
express-spell-check
rails-weather-dashboard
go-barcode-generator
```

Place the directory at the **repo root** (not nested).

---

## Required Files per Project

Every project must include:

| File | Purpose |
|------|---------|
| `README.md` | What it does, prerequisites, setup, example output |
| `.env.example` | Template with `REQUIEMS_API=your_api_key_here` |
| `.gitignore` | Exclude `.env`, build artifacts, `node_modules`, `.venv`, etc. |

---

## Environment & Secrets

- **Never commit API keys.**
- The root `.env` holds `REQUIEMS_API` and is the single source of truth for local development.
- Projects should either read `../.env` (relative path) or document that users copy it:
  ```bash
  cp ../.env .env
  ```
- Always provide `.env.example` so contributors know what variables are needed:
  ```
  REQUIEMS_API=your_api_key_here
  ```

---

## Tooling Standards

### Node.js / TypeScript — use pnpm

**Always pnpm. Never npm, yarn, or bun.**

```bash
# Create a new project
pnpm create next-app my-project
# or
pnpm init

# Install dependencies
pnpm install
pnpm add axios dotenv

# Run
pnpm dev
pnpm build
pnpm start
```

`package.json` scripts should use `pnpm` in any documented commands.

### Python — use uv + ruff

**Always uv for project management. Always ruff for linting and formatting.**
Never pip, poetry, pipenv, flake8, black, isort, or pylint.

```bash
# Create a new project
uv init my-project
cd my-project

# Add dependencies
uv add requests python-dotenv

# Run the project
uv run python main.py

# Lint
ruff check .

# Format
ruff format .
```

The `pyproject.toml` generated by `uv init` already supports ruff configuration:

```toml
[tool.ruff]
line-length = 88
target-version = "py312"

[tool.ruff.lint]
select = ["E", "F", "I"]
```

### Other Languages

| Language | Package manager | Lint/format |
|----------|----------------|-------------|
| Go | `go mod` | `go fmt`, `golangci-lint` |
| Ruby | `bundler` | `rubocop` |
| Rust | `cargo` | `rustfmt`, `clippy` |

## Walkthrough: New Node.js Example (pnpm)

This example builds a simple server that calls the QR code endpoint.

```bash
# 1. Create and enter the directory
mkdir express-qr-code && cd express-qr-code

# 2. Init with pnpm
pnpm init

# 3. Add dependencies
pnpm add express dotenv

# 4. Create .env.example
echo "REQUIEMS_API=your_api_key_here" > .env.example

# 5. Copy root env for local development
cp ../.env .env
```

**`index.js`**:
```javascript
import 'dotenv/config'
import express from 'express'

const app = express()
app.use(express.json())

app.post('/qr', async (req, res) => {
  const { text } = req.body
  const response = await fetch('https://api.requiems.xyz/v1/tech/qr', {
    method: 'POST',
    headers: {
      'requiems-api-key': process.env.REQUIEMS_API,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({ text })
  })
  const { data } = await response.json()
  res.json(data)
})

app.listen(3000, () => console.log('Listening on http://localhost:3000'))
```

**`.gitignore`**:
```
node_modules/
.env
dist/
```

**`package.json`** scripts:
```json
{
  "scripts": {
    "start": "node index.js",
    "dev": "node --watch index.js"
  }
}
```

## Walkthrough: New Python Example (uv + ruff)

This example calls the spell check endpoint from the command line.

```bash
# 1. Create project with uv
uv init python-spell-check
cd python-spell-check

# 2. Add dependencies
uv add requests python-dotenv

# 3. Create .env.example
echo "REQUIEMS_API=your_api_key_here" > .env.example

# 4. Copy root env
cp ../.env .env
```

**`main.py`**:
```python
import os
import sys
import requests
from dotenv import load_dotenv

load_dotenv()

API_KEY = os.environ["REQUIEMS_API"]
BASE_URL = "https://api.requiems.xyz"


def spell_check(text: str) -> dict:
    response = requests.post(
        f"{BASE_URL}/v1/text/spellcheck",
        headers={"requiems-api-key": API_KEY, "Content-Type": "application/json"},
        json={"text": text},
    )
    response.raise_for_status()
    return response.json()["data"]


if __name__ == "__main__":
    text = " ".join(sys.argv[1:]) or "Ths is a tset sentance."
    result = spell_check(text)
    print(result)
```

**Run it:**
```bash
uv run python main.py "Ths is a tset sentance."
```

**Lint and format:**
```bash
ruff check .
ruff format .
```

**`.gitignore`**:
```
.env
.venv/
__pycache__/
*.pyc
.ruff_cache/
```

## Project README Template

Each project's `README.md` should follow this structure:

```markdown
# {project-name}

One sentence describing what this example demonstrates and which API it uses.

## What it does

Short description (2-3 sentences) of the user-facing functionality.

## Prerequisites

- Node.js 20+ / Python 3.12+ / etc.
- pnpm / uv
- A Requiems API key ([get one here](https://requiems.xyz))

## Setup

\```bash
# Install dependencies
pnpm install   # or: uv sync

# Copy env file
cp ../.env .env   # or add REQUIEMS_API=your_key to .env

# Run
pnpm dev   # or: uv run python main.py
\```

## Example Output

\```
(paste real output here)
\```

## API Used

- [`POST /v1/{endpoint}`](https://requiems.xyz/en/apis) — brief description
```

---

## Checklist Before Submitting

- [ ] Project runs from scratch: `pnpm install && pnpm dev` or `uv run python main.py`
- [ ] No API key committed (check with `git diff --cached`)
- [ ] `.env.example` present with all required variables
- [ ] `.gitignore` excludes `.env`, build artifacts, and dependency directories
- [ ] Project `README.md` includes example output
- [ ] Code is linted (`ruff check .` / biome / etc. — no errors)
- [ ] Root `readme.md` projects table updated with a new row
- [ ] Directory name follows `{framework}-{api-feature}` convention

## Finding the Right Endpoint

API docs are available two ways:

- **Online**: https://requiems.xyz/en/apis
- **Local**: `../requiems-api/docs/apis/{category}/{endpoint}.md`

Categories: `ai-computer-vision`, `animals`, `email`, `entertainment`, `finance`, `health-wellness`, `internet-technology`, `miscellaneous`, `places`, `text`, `transportation`, `weather`
